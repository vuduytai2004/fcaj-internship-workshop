### 0. Tạo IAM Groups và IAM Users cho workshop

Trước khi tạo các service role cho hệ thống, hãy tạo các IAM group để quản lý quyền truy cập AWS Console theo vai trò. Cách này giúp bạn không phải gắn policy trực tiếp vào từng user.

{{% notice warning %}}
Các policy như `AdministratorAccess` và `PowerUserAccess` chỉ phù hợp cho môi trường workshop/lab có kiểm soát. Với môi trường production, hãy thay bằng custom policy giới hạn đúng tài nguyên và hành động cần thiết.
{{% /notice %}}

### - Role 1: `docuflow-dev-security-upload-url-role` (Lambda sinh URL)
**Mục đích**: Giúp API Gateway gọi Lambda tạo link Presigned URL để Front-end đẩy file lên S3 Raw và ghi nhận thông tin ban đầu vào cơ sở dữ liệu.
* **Các bước thực hiện**:
  1. Vào dịch vụ **IAM** ➔ **Policies** ➔ Chọn **Create policy** (Chọn tab **JSON**).
  2. Dán đoạn JSON Policy sau:
     ```json
     {
         "Version": "2012-10-17",
         "Statement": [
             {
                 "Sid": "AllowPutObjectToRawBucket",
                 "Effect": "Allow",
                 "Action": "s3:PutObject",
                 "Resource": "arn:aws:s3:::docuflow-dev-raw-*"
             },
             {
                 "Sid": "AllowWriteToDynamoDB",
                 "Effect": "Allow",
                 "Action": [
                     "dynamodb:PutItem",
                     "dynamodb:UpdateItem"
                 ],
                 "Resource": "arn:aws:dynamodb:*:*:table/docuflow-dev-documents-table"
             }
         ]
     }
     ```
     ![image1.png](/images/5-Workshop/5.3-prepare-project-foundation/5.3.1-iam/image1.png)
  3. Bấm **Next**. Đặt tên Policy Name: `docuflow-dev-ingestion-s3-raw-access-policy`.


  4. Điền Description: `Allows Lambda function to generate S3 presigned URLs for raw document upload and initialize metadata in DynamoDB table.`
     ![image2.png](/images/5-Workshop/5.3-prepare-project-foundation/5.3.1-iam/image2.png)
  5. Bấm **Create policy**.
  6. Vào mục **Roles** ➔ Chọn **Create role** (Chọn Trusted Entity Type: **AWS service** ➔ Service or use case: **Lambda**).
  ![image3.png](/images/5-Workshop/5.3-prepare-project-foundation/5.3.1-iam/image3.png)
  7. Bấm **Next**.
  8. Gắn Policy: Tích chọn `docuflow-dev-ingestion-s3-raw-access-policy` và policy mặc định hệ thống `AWSLambdaBasicExecutionRole` (để ghi log).
  ![image4.png](/images/5-Workshop/5.3-prepare-project-foundation/5.3.1-iam/image4.png)
  ![image5.png](/images/5-Workshop/5.3-prepare-project-foundation/5.3.1-iam/image5.png)
  9. Bấm **Next**. Đặt Role Name: `docuflow-dev-security-upload-url-role`.
  ![image6.png](/images/5-Workshop/5.3-prepare-project-foundation/5.3.1-iam/image6.png)
  10. Bấm **Create role**.

### - Role 2: `docuflow-dev-security-job-starter-role` (Quyền cho Lambda đọc SQS kích hoạt Workflow)
**Mục đích**: Giúp Lambda nhận tin nhắn từ hàng đợi SQS để lấy ID file, sau đó kích hoạt luồng xử lý tự động của Step Functions.
* **Các bước thực hiện**:
  1. Vào dịch vụ **IAM** ➔ **Policies** ➔ Chọn **Create policy** (Chọn tab **JSON**).
  2. Dán đoạn JSON Policy sau:
     ```json
     {
         "Version": "2012-10-17",
         "Statement": [
             {
                 "Sid": "AllowStartStepFunctions",
                 "Effect": "Allow",
                 "Action": "states:StartExecution",
                 "Resource": "arn:aws:states:*:*:stateMachine:docuflow-dev-*"
             },
             {
                 "Sid": "AllowManageSQSMessages",
                 "Effect": "Allow",
                 "Action": [
                     "sqs:ReceiveMessage",
                     "sqs:DeleteMessage",
                     "sqs:GetQueueAttributes"
                 ],
                 "Resource": "arn:aws:sqs:*:*:docuflow-dev-*"
             }
         ]
     }
     ```
     ![image8.png](/images/5-Workshop/5.3-prepare-project-foundation/5.3.1-iam/image8.png)
  3. Bấm **Next**. Đặt tên Policy Name: `docuflow-dev-workflow-trigger-policy`.

     ![image9.png](/images/5-Workshop/5.3-prepare-project-foundation/5.3.1-iam/image9.png)
  5. Bấm **Create policy**.

### - Role 3: `docuflow-dev-workflow-stepfunctions-role` (Quyền dành riêng cho Bộ máy điều phối Step Functions)
**Mục đích**: Cấp quyền cho dịch vụ Step Functions có thể gọi thực thi tuần tự các hàm Lambda xử lý và ghi log tiến trình vận hành.
* **Các bước thực hiện**:
  1. Vào dịch vụ **IAM** ➔ **Policies** ➔ Chọn **Create policy** (Chọn tab **JSON**).
  2. Dán đoạn JSON Policy sau:
     ```json
     {
         "Version": "2012-10-17",
         "Statement": [
             {
                 "Sid": "AllowStepFunctionToCallLambda",
                 "Effect": "Allow",
                 "Action": "lambda:InvokeFunction",
                 "Resource": "arn:aws:lambda:*:*:function:docuflow-dev-*"
             },
             {
                 "Sid": "AllowStepFunctionsLogging",
                 "Effect": "Allow",
                 "Action": [
                     "logs:CreateLogDelivery",
                     "logs:PutLogEvents",
                     "logs:GetLogDelivery",
                     "logs:UpdateLogDelivery",
                     "logs:DeleteLogDelivery",
                     "logs:ListLogDeliveries"
                 ],
                 "Resource": "*"
             }
         ]
     }
     ```
     ![image16.png](/images/5-Workshop/5.3-prepare-project-foundation/5.3.1-iam/image16.png)
  3. Bấm **Next**. Đặt tên Policy Name: `docuflow-dev-workflow-orchestration-policy`.
  4. Điền Description: `Allows Step Functions to orchestrate workflow by invoking core processing Lambdas and logging deliveries.`
     ![image17.png](/images/5-Workshop/5.3-prepare-project-foundation/5.3.1-iam/image17.png)
  5. Bấm **Create policy**.
  ![image18.png](/images/5-Workshop/5.3-prepare-project-foundation/5.3.1-iam/image18.png)
  6. Vào mục **Roles** ➔ Chọn **Create role** (Chọn Trusted Entity Type: **Custom trust policy**).
  
  7. Dán đoạn Trust Policy định nghĩa dịch vụ Step Functions sau vào:
     ```json
     {
       "Version": "2012-10-17",
       "Statement": [{ "Effect": "Allow", "Principal": { "Service": "states.amazonaws.com" }, "Action": "sts:AssumeRole" }]
     }
     ```
     ![image19.png](/images/5-Workshop/5.3-prepare-project-foundation/5.3.1-iam/image19.png)
  8. Bấm **Next**. Gắn Policy: Tích chọn `docuflow-dev-workflow-orchestration-policy`.
  ![image20.png](/images/5-Workshop/5.3-prepare-project-foundation/5.3.1-iam/image20.png)
  9. Bấm **Next**. Đặt Role Name: `docuflow-dev-workflow-stepfunctions-role`.
  10. Điền Description: `Service role for Step Functions state machine to orchestrate the AI document pipeline.`
  ![image21.png](/images/5-Workshop/5.3-prepare-project-foundation/5.3.1-iam/image21.png)
  11. Bấm **Create role**.
  ![image22.png](/images/5-Workshop/5.3-prepare-project-foundation/5.3.1-iam/image22.png)

### - Role 5: `docuflow-dev-security-ai-proxy-role` (Quyền cho Lambda gọi External AI ngoài AWS)

* **Các bước thực hiện**:
  1. Vào dịch vụ **IAM** ➔ **Policies** ➔ Chọn **Create policy** (JSON).
  2. Dán đoạn JSON Policy sau (thay đổi ARN tương ứng nếu cần thiết):
     ```json
     {
         "Version": "2012-10-17",
         "Statement": [
             {
                 "Sid": "AllowAIProxyToGetSecret",
                 "Effect": "Allow",
                 "Action": "secretsmanager:GetSecretValue",
                 "Resource": "arn:aws:secretsmanager:ap-southeast-1:*:secret:docuflow-dev-external-ai-api-key-*"
             },
             {
                 "Sid": "AllowAIProxyToUseKMS",
                 "Effect": "Allow",
                 "Action": [
                     "kms:Decrypt",
                     "kms:DescribeKey"
                 ],
                 "Resource": "arn:aws:kms:*:*:alias/docuflow-dev-main-key"
             }
         ]
     }
     ```

### - Role 6: `docuflow-dev-data-lambda-role` (Quyền cho Lambda CRUD Dữ liệu hiển thị UI)
**Mục đích**: Cho phép các hàm Lambda API thực hiện CRUD trên DynamoDB và tải tệp từ các S3 bucket để hiển thị Frontend.

  1. Vào dịch vụ **IAM** ➔ **Policies** ➔ Chọn **Create policy** (JSON).
  2. Dán đoạn JSON Policy sau:
     ```json
     {
         "Version": "2012-10-17",
         "Statement": [
             {
                 "Sid": "AllowReadWriteDynamoDBOnly",
                 "Effect": "Allow",
                 "Action": [
                     "dynamodb:GetItem",
                     "dynamodb:Query",
                     "dynamodb:UpdateItem",
                     "dynamodb:Scan",
                     "dynamodb:DeleteItem"
                 ],
                 "Resource": [
                     "arn:aws:dynamodb:*:*:table/docuflow-dev-documents-table",
                     "arn:aws:dynamodb:*:*:table/docuflow-dev-documents-table/index/*"
                 ]
             },
             {
                 "Sid": "AllowReadS3ProcessedBucket",
                 "Effect": "Allow",
                 "Action": [
                     "s3:GetObject",
                     "s3:DeleteObject",
                     "s3:ListBucket"
                 ],
                 "Resource": [
                     "arn:aws:s3:::docuflow-dev-processed-*/*",
                     "arn:aws:s3:::docuflow-dev-raw-*/*",
                     "arn:aws:s3:::docuflow-dev-processed-*",
                     "arn:aws:s3:::docuflow-dev-raw-*"

             }
         ]
     }
     ```
