# EVENT SUMMARY REPORT: FCAJ COMMUNITY DAY ON MAY 09, 2026

### Event Objectives
The event is a meetup for the tech community (especially Cloud and software). The main purpose is to share professional knowledge, provide career orientation for students/freshers, and create a safe environment for young people to practice presentation skills and share knowledge without fear of judgment.

### Speakers
- **Mr. Huynh Hoang Long**: Shared on how to "hack" the brain to create motivation for learning.
- **Mr. Nguyen Tuan Thinh**: Presented on Prompt Engineering and Serverless architecture design on AWS.
- **Mr. Khang**: Solutions Architect at Cloud Kinetics, shared about career orientation and mindset in the AI era.
- **Ms. Thao**: Software Developer at VIB international bank, presented the methodology of using AI in software development.

### Key Highlights
The event revolved around 4 main topics:
1. **"Hack" the brain to get addicted to learning**: The speaker explained how the brain reacts to Dopamine (reward anticipation) to create learning tips such as: breaking down knowledge (e.g., studying one AWS service every day), creating a reward draw box, maintaining a learning streak, and applying the 2-minute rule.
2. **Prompt Engineering & AWS Skills**: Guided how to optimize AI prompts (clearly defining roles, contexts, output formats) to avoid AI "hallucination". Also demoed a prompt optimization project using AWS Serverless infrastructure (S3, CloudFront, API Gateway, Lambda, DynamoDB, Bedrock).
3. **Career orientation in the AI era**: Emphasized that AI does not replace humans but only amplifies capabilities: if you are good, AI makes you 10x better; if you lack foundation, AI will expose that. The speaker particularly highlighted "integrity" and the mindset of always asking "Why" when solving problems.
4. **BMX method in programming with AI**: Presented the method of breaking down a project into roles for AI agents (PM, Architect, Scrum Master, Developer). Instead of asking AI to write code directly which easily generates garbage code, this method forces AI to write documents/stories first, then code in small chunks to control quality.

### Key Takeaways
- **Mindset**: Understood that the root of laziness is not nature, but because the brain is not satisfied with curiosity and quick rewards. Learned a profound lesson that employers do not just look for someone who does things right, but someone who understands the foundation and can explain the reasons behind their decisions.
- **Soft skills**: Drew the importance of daring to do, daring to be wrong, and maintaining integrity at work, looking at a career in the long term instead of short-term benefits.
- **Technical**: Grasped how to systematically divide the workflow when developing software with AI to prevent AI from losing context (context window).

### Applying to Work
- **Create working/learning habits**: Apply a self-rewarding system and record streaks on the calendar to maintain daily learning of new technologies without getting overwhelmed. Immediately resolve tasks that take less than 2 minutes.
- **Communicate more effectively with AI**: Use the standard Prompt formula (Role + Task + Context + Format + Constraint) for AI to return accurate results for daily programming or writing tasks.
- **Manage Code projects with AI**: Apply the mindset of the BMX method: Always write clear documents, break down Epics/Stories before asking tools like Cursor/Copilot to write code to ensure the generated code does not have architectural flaws.
