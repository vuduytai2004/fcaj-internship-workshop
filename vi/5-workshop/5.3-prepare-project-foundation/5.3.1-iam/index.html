Cấu hình IAM Groups và Roles

Để tuân thủ nguyên tắc phân quyền tối thiểu (Least Privilege), chúng ta sẽ tạo IAM Groups cho người dùng vận hành workshop, sau đó thiết kế các IAM Policy và Role chuyên biệt cho từng dịch vụ Lambda và Step Functions trong hệ thống.

0. Tạo IAM Groups và IAM Users cho workshop

Trước khi tạo các service role cho hệ thống, hãy tạo các IAM group để quản lý quyền truy cập AWS Console theo vai trò. Cách này giúp bạn không phải gắn policy trực tiếp vào từng user.

Các policy như AdministratorAccess và PowerUserAccess chỉ phù hợp cho môi trường workshop/lab có kiểm soát. Với môi trường production, hãy thay bằng custom policy giới hạn đúng tài nguyên và hành động cần thiết.

  1. Vào dịch vụ IAM -> User groups -> Create group.

  2. Tạo group quản trị:

    • User group name: DocuFlow-Admins
    • Permissions policy: AdministratorAccess

  3. Tạo group dành cho developer:

    • User group name: DocuFlow-Developers
    • Permissions policy: PowerUserAccess

  4. Tạo group chỉ đọc:

    • User group name: DocuFlow-ReadOnly
    • Permissions policy: ReadOnlyAccess

  5. Tạo admin user mẫu:

    • Vào IAM -> Users -> Create user.
    • Nhập user name, ví dụ docuflow-tra.
    • Chọn Provide user access to the AWS Management Console.
    • Chọn Autogenerated password và bật yêu cầu đổi mật khẩu ở lần đăng nhập đầu tiên.
    • Ở bước Set permissions, chọn Add user to group và gắn user vào DocuFlow-Admins.

  6. Tạo các user mẫu cho workshop:

    • Lặp lại thao tác Create user cho các user developer như docuflow-tinh, docuflow-tai, docuflow-duong.
    • Nếu cần user chỉ xem, tạo thêm user read-only, ví dụ docuflow-loc.
    • Bật quyền truy cập AWS Management Console.
    • Dùng mật khẩu tự sinh và yêu cầu đổi mật khẩu khi đăng nhập lần đầu.

  7. Ở bước Set permissions, chọn Add user to group. Gắn user developer vào DocuFlow-Developers; nếu tạo user chỉ xem, gắn user đó vào DocuFlow-ReadOnly.

  8. Sau khi tạo xong, quay lại IAM -> User groups để xác nhận các group đã có user và policy đúng như mong muốn.

- Role 1: docuflow-dev-security-upload-url-role (Lambda sinh URL)

Mục đích: Giúp API Gateway gọi Lambda tạo link Presigned URL để Front-end đẩy file lên S3 Raw và ghi nhận thông tin ban đầu vào cơ sở dữ liệu.

  • Các bước thực hiện:

    1. Vào dịch vụ IAM ➔ Policies ➔ Chọn Create policy (Chọn tab JSON).

    2. Dán đoạn JSON Policy sau:

      {
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "AllowPutObjectToRawBucket",
            "Effect": "Allow",
            "Action": "s3:PutObject",
            "Resource": "arn:aws:s3:::docuflow-dev-raw-*"
        },
        {
            "Sid": "AllowWriteToDynamoDB",
            "Effect": "Allow",
            "Action": [
                "dynamodb:PutItem",
                "dynamodb:UpdateItem"
            ],
            "Resource": "arn:aws:dynamodb:*:*:table/docuflow-dev-documents-table"
        }
    ]
}

    3. Bấm Next. Đặt tên Policy Name: docuflow-dev-ingestion-s3-raw-access-policy.

    4. Điền Description: Allows Lambda function to generate S3 presigned URLs for raw document upload and initialize metadata in DynamoDB table.

    5. Bấm Create policy.

    6. Vào mục Roles ➔ Chọn Create role (Chọn Trusted Entity Type: AWS service ➔ Service or use case: Lambda).

    7. Bấm Next.

    8. Gắn Policy: Tích chọn docuflow-dev-ingestion-s3-raw-access-policy và policy mặc định hệ thống AWSLambdaBasicExecutionRole (để ghi log).

    9. Bấm Next. Đặt Role Name: docuflow-dev-security-upload-url-role.

    10. Bấm Create role.

- Role 2: docuflow-dev-security-job-starter-role (Quyền cho Lambda đọc SQS kích hoạt Workflow)

Mục đích: Giúp Lambda nhận tin nhắn từ hàng đợi SQS để lấy ID file, sau đó kích hoạt luồng xử lý tự động của Step Functions.

  • Các bước thực hiện:

    1. Vào dịch vụ IAM ➔ Policies ➔ Chọn Create policy (Chọn tab JSON).

    2. Dán đoạn JSON Policy sau:

      {
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "AllowStartStepFunctions",
            "Effect": "Allow",
            "Action": "states:StartExecution",
            "Resource": "arn:aws:states:*:*:stateMachine:docuflow-dev-*"
        },
        {
            "Sid": "AllowManageSQSMessages",
            "Effect": "Allow",
            "Action": [
                "sqs:ReceiveMessage",
                "sqs:DeleteMessage",
                "sqs:GetQueueAttributes"
            ],
            "Resource": "arn:aws:sqs:*:*:docuflow-dev-*"
        }
    ]
}

    3. Bấm Next. Đặt tên Policy Name: docuflow-dev-workflow-trigger-policy.

    4. Điền Description: Grants permissions to process messages from SQS queue and trigger Step Functions workflow executions.

    5. Bấm Create policy.

    6. Vào mục Roles ➔ Chọn Create role (Chọn Trusted Entity Type: AWS service ➔ Service or use case: Lambda).

    7. Bấm Next.

    8. Gắn Policy: Tích chọn docuflow-dev-workflow-trigger-policy và AWSLambdaBasicExecutionRole.

    9. Bấm Next. Đặt Role Name: docuflow-dev-security-job-starter-role.

    10. Bấm Create role.

- Role 3: docuflow-dev-workflow-stepfunctions-role (Quyền dành riêng cho Bộ máy điều phối Step Functions)

Mục đích: Cấp quyền cho dịch vụ Step Functions có thể gọi thực thi tuần tự các hàm Lambda xử lý và ghi log tiến trình vận hành.

  • Các bước thực hiện:

    1. Vào dịch vụ IAM ➔ Policies ➔ Chọn Create policy (Chọn tab JSON).

    2. Dán đoạn JSON Policy sau:

      {
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "AllowStepFunctionToCallLambda",
            "Effect": "Allow",
            "Action": "lambda:InvokeFunction",
            "Resource": "arn:aws:lambda:*:*:function:docuflow-dev-*"
        },
        {
            "Sid": "AllowStepFunctionsLogging",
            "Effect": "Allow",
            "Action": [
                "logs:CreateLogDelivery",
                "logs:PutLogEvents",
                "logs:GetLogDelivery",
                "logs:UpdateLogDelivery",
                "logs:DeleteLogDelivery",
                "logs:ListLogDeliveries"
            ],
            "Resource": "*"
        }
    ]
}

    3. Bấm Next. Đặt tên Policy Name: docuflow-dev-workflow-orchestration-policy.

    4. Điền Description: Allows Step Functions to orchestrate workflow by invoking core processing Lambdas and logging deliveries.

    5. Bấm Create policy.

    6. Vào mục Roles ➔ Chọn Create role (Chọn Trusted Entity Type: Custom trust policy).

    7. Dán đoạn Trust Policy định nghĩa dịch vụ Step Functions sau vào:

      {
  "Version": "2012-10-17",
  "Statement": [{ "Effect": "Allow", "Principal": { "Service": "states.amazonaws.com" }, "Action": "sts:AssumeRole" }]
}

    8. Bấm Next. Gắn Policy: Tích chọn docuflow-dev-workflow-orchestration-policy.

    9. Bấm Next. Đặt Role Name: docuflow-dev-workflow-stepfunctions-role.

    10. Điền Description: Service role for Step Functions state machine to orchestrate the AI document pipeline.

    11. Bấm Create role.

- Role 4: docuflow-dev-ai-textract-lambda-role (Quyền cho Lambda xử lý trích xuất văn bản Amazon Textract)

Mục đích: Cho phép Lambda lấy file từ S3 Raw đem đi trích xuất qua dịch vụ Textract và lưu kết quả thô sang S3 Processed Bucket.

  • Các bước thực hiện:

    1. Vào dịch vụ IAM ➔ Policies ➔ Chọn Create policy (JSON).
    2. Dán đoạn JSON Policy sau:
      {
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "AllowReadRawBucket",
            "Effect": "Allow",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::docuflow-dev-raw-*"
        },
        {
            "Sid": "AllowWriteProcessedBucket",
            "Effect": "Allow",
            "Action": "s3:PutObject",
            "Resource": "arn:aws:s3:::docuflow-dev-processed-*"
        },
        {
            "Sid": "AllowCallTextractAI",
            "Effect": "Allow",
            "Action": "textract:AnalyzeExpense",
            "Resource": "*"
        }
    ]
}
    3. Bấm Next. Đặt tên Policy Name: docuflow-dev-ai-textract-invoke-policy.
    4. Điền Description: Allows Lambda to retrieve raw documents from S3, invoke Amazon Textract AnalyzeExpense API, and save results.
    5. Bấm Create policy.
    6. Vào mục Roles ➔ Chọn Create role (Trusted Entity Type: AWS service ➔ Service: Lambda).
    7. Bấm Next.
    8. Gắn Policy: Tích chọn docuflow-dev-ai-textract-invoke-policy và AWSLambdaBasicExecutionRole, AmazonS3ExpressFullAccess, AmazonTextractFullAccess.

    9. Bấm Next. Đặt Role Name: docuflow-dev-ai-textract-lambda-role. 10. Điền Description: Execution role for extraction lambda to read raw invoices, run Textract OCR, and write thô data. 11. Bấm Create role.

- Role 5: docuflow-dev-security-ai-proxy-role (Quyền cho Lambda gọi External AI ngoài AWS)

Mục đích: Chốt chặn an ninh tối cao. Chỉ duy nhất hàm AI Proxy này được phép đọc mã Token (API Key) cất trong Secrets Manager để kết nối với mô hình AI bên ngoài.

  • Các bước thực hiện:
    1. Vào dịch vụ IAM ➔ Policies ➔ Chọn Create policy (JSON).
    2. Dán đoạn JSON Policy sau (thay đổi ARN tương ứng nếu cần thiết):
      {
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "AllowAIProxyToGetSecret",
            "Effect": "Allow",
            "Action": "secretsmanager:GetSecretValue",
            "Resource": "arn:aws:secretsmanager:ap-southeast-1:*:secret:docuflow-dev-external-ai-api-key-*"
        },
        {
            "Sid": "AllowAIProxyToUseKMS",
            "Effect": "Allow",
            "Action": [
                "kms:Decrypt",
                "kms:DescribeKey"
            ],
            "Resource": "arn:aws:kms:*:*:alias/docuflow-dev-main-key"
        }
    ]
}
    3. Bấm Next. Đặt tên Policy Name: docuflow-dev-ai-secret-read-policy.
    4. Điền Description: Strictly restricts access to External AI API credentials stored within AWS Secrets Manager.
    5. Bấm Create policy.
    6. Vào mục Roles ➔ Chọn Create role (Trusted Entity Type: AWS service ➔ Service: Lambda).
    7. Bấm Next.
    8. Gắn Policy: Tích chọn docuflow-dev-ai-secret-read-policy và AWSLambdaBasicExecutionRole.
    9. Bấm Next. Đặt Role Name: docuflow-dev-security-ai-proxy-role.
    10. Bấm Create role.

- Role 6: docuflow-dev-data-lambda-role (Quyền cho Lambda CRUD Dữ liệu hiển thị UI)

Mục đích: Cho phép các hàm Lambda API thực hiện CRUD trên DynamoDB và tải tệp từ các S3 bucket để hiển thị Frontend.

  • Các bước thực hiện:
    1. Vào dịch vụ IAM ➔ Policies ➔ Chọn Create policy (JSON).
    2. Dán đoạn JSON Policy sau:
      {
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "AllowReadWriteDynamoDBOnly",
            "Effect": "Allow",
            "Action": [
                "dynamodb:GetItem",
                "dynamodb:Query",
                "dynamodb:UpdateItem",
                "dynamodb:Scan",
                "dynamodb:DeleteItem"
            ],
            "Resource": [
                "arn:aws:dynamodb:*:*:table/docuflow-dev-documents-table",
                "arn:aws:dynamodb:*:*:table/docuflow-dev-documents-table/index/*"
            ]
        },
        {
            "Sid": "AllowReadS3ProcessedBucket",
            "Effect": "Allow",
            "Action": [
                "s3:GetObject",
                "s3:DeleteObject",
                "s3:ListBucket"
            ],
            "Resource": [
                "arn:aws:s3:::docuflow-dev-processed-*/*",
                "arn:aws:s3:::docuflow-dev-raw-*/*",
                "arn:aws:s3:::docuflow-dev-processed-*",
                "arn:aws:s3:::docuflow-dev-raw-*"
            ]
        }
    ]
}
    3. Bấm Next. Đặt tên Policy Name: docuflow-dev-data-dynamodb-access-policy. 4. Điền Description: Allows standard query, scanning, and metadata update operations against the core DynamoDB documents table. 5. Bấm Create policy. 6. Vào mục Roles ➔ Chọn Create role (Trusted Entity Type: AWS service ➔ Service: Lambda). 7. Bấm Next. 8. Gắn Policy: Tích chọn docuflow-dev-data-dynamodb-access-policy và AWSLambdaBasicExecutionRole. 9. Bấm Next. Đặt Role Name: docuflow-dev-data-lambda-role.
  1. Điền Description: Execution role for metadata/data lambda to fetch and modify documents inside DynamoDB.
  2. Bấm Create role.

    9. Bấm Next. Đặt Role Name: docuflow-dev-notification-lambda-role.
    10. Điền Description: Execution role for notification component to broadcast system alert messages via SNS.
    11. Bấm Create role.

- Role 8: docuflow-dev-workflow-validate-lambda-role (Quyền cho Lambda kiểm tra dữ liệu đầu vào)

            "Effect": "Allow",

                "s3:GetObject",
                "s3:GetObjectAttributes"
            ],
            "Resource": "arn:aws:s3:::docuflow-dev-raw-*"
        },
        {
            "Sid": "AllowUpdateStatusToDynamoDB",
            "Effect": "Allow",
            "Action": [
                "dynamodb:UpdateItem",
                "dynamodb:GetItem"
            ],
            "Resource": "arn:aws:dynamodb:*:*:table/docuflow-dev-documents-table"
        }
    ]
}
    3. Bấm Next. Đặt tên Policy Name: docuflow-dev-ingestion-validation-policy.
    4. Điền Description: Allows Validate Lambda to check file metadata in S3 Raw and update initial processing state in DynamoDB.
    5. Bấm Create policy.
    6. Vào mục Roles ➔ Chọn Create role (Trusted Entity Type: AWS service ➔ Service: Lambda).
    7. Bấm Next.
    8. Gắn Policy: Tích chọn docuflow-dev-ingestion-validation-policy và AWSLambdaBasicExecutionRole.
    9. Bấm Next. Đặt Role Name: docuflow-dev-workflow-validate-lambda-role.
    10. Điền Description: Execution role for validation lambda to inspect uploaded documents and update state.
    11. Bấm Create role.

- Role 9: docuflow-dev-ai-confidence-status-lambda-role (Quyền cho Lambda xử lý Logic Trạng thái & Đánh giá Độ tin cậy)

Mục đích: Sau khi AI Proxy trả về kết quả, hàm Confidence + Status Lambda sẽ chịu trách nhiệm chấm điểm, phân loại trạng thái thành EXTRACTED hoặc REVIEW_REQUIRED, đồng thời thực hiện lưu kết quả cuối cùng (result.json) vào S3 Processed Bucket và cập nhật bảng DynamoDB.

  • Các bước thực hiện:
    1. Vào dịch vụ IAM ➔ Policies ➔ Chọn Create policy (JSON).
    2. Dán đoạn JSON Policy sau:
      {
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "AllowWriteToProcessedBucket",
